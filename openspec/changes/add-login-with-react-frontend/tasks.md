# 登录功能任务分解（含单点登录原型）

## 单点登录（SSO）原理说明
- 单点登录（SSO）允许用户在一个系统登录后，无需再次登录即可访问多个受信任的子系统。
- 实现方式：
  1. 用户访问任一子系统，未登录时被重定向到统一认证中心（本项目用 /auth/login 实现）。
  2. 用户在认证中心登录成功后，认证中心生成 token（如 JWT），并将 token 返回给前端。
  3. 前端将 token 存储（如 localStorage），后续访问所有子系统时在请求头携带 token。
  4. 各子系统收到请求后，统一校验 token，校验通过则允许访问。
- 本项目通过模拟多个前端页面和统一后端接口，演示 SSO 流程。
